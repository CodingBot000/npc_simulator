# NPC Simulator

웹 기반 MMORPG형 NPC 엔진 프로토타입.

이 프로젝트는 아래 3가지를 조합한 사용자 경험을 목표로 한다.

- 관계 중심 소셜 시뮬레이터
- 퀘스트 협상형 NPC 인터페이스
- 얇은 감독자 모드

## 한 줄 컨셉

사용자는 마을이나 길드 거점에서 NPC와 대화하고, 설득하고, 거래하고, 부탁을 수행한다.  
NPC는 사용자를 기억하고, 다른 NPC와의 관계와 최근 사건을 반영해 태도와 발화를 바꾼다.

## 왜 이 조합이 좋은가

- 관계 중심 구조는 NPC가 살아 있는 세계 구성원처럼 느껴지게 한다.
- 퀘스트 협상 구조는 대화에 명확한 목표를 부여해 재미를 만든다.
- 감독자 모드는 포트폴리오에서 엔진의 기억, 의도, 행동 선택 과정을 설명하기 좋다.

## 핵심 사용자 경험

사용자는 웹페이지에 접속하면 작은 마을 허브에 들어온다.

- 각 NPC는 현재 위치, 감정, 관심사, 플레이어에 대한 태도를 가진다.
- 사용자는 NPC를 클릭해 대화하거나 부탁을 하거나 정보를 캐낼 수 있다.
- 같은 질문이라도 NPC의 기억과 현재 목표에 따라 다른 반응이 나온다.
- 특정 NPC와의 상호작용은 다른 NPC 관계에도 영향을 준다.
- 대화 결과는 퀘스트, 평판, 신뢰도, 소문 형태로 세계에 남는다.

## 메인 루프

1. 사용자가 NPC를 선택한다.
2. NPC의 현재 상태와 최근 사건이 요약되어 보인다.
3. 사용자는 자유 입력 또는 행동 버튼으로 상호작용한다.
4. NPC는 기억 검색, 관계 상태, 의도 판단을 바탕으로 응답한다.
5. 결과가 관계도, 퀘스트 상태, 월드 이벤트 로그에 반영된다.
6. 사용자는 다른 NPC에게 이동해 변화된 반응을 확인한다.

## 화면 구성

### 1. 월드 허브

- 중앙: 마을, 길드 홀, tavern 같은 허브 장면
- 좌측: NPC 목록과 현재 상태
- 우측: 선택한 NPC 정보 카드
- 하단: 월드 이벤트 로그

### 2. NPC 상호작용 패널

- 대화 기록
- 자유 입력창
- 행동 버튼: 설득, 거래, 질문, 부탁, 압박, 공감
- 현재 퀘스트/협상 상태

### 3. 관계/파장 패널

- 플레이어와 NPC 간 신뢰도
- NPC끼리의 관계 변화
- 최근 소문 또는 사건 전파 로그

### 4. 얇은 감독자 모드

기본 화면을 해치지 않는 선에서 우측 접이식 패널로 제공한다.

- 회수된 기억 3~5개
- 현재 감정 상태
- 현재 의도
- 후보 행동 2~3개
- 최종 선택 이유 한 줄

## MVP 범위

### 반드시 포함

- 허브 공간 1개
- NPC 3명
- 플레이어별 기억
- NPC 간 간단한 관계값
- 퀘스트 2~3개
- 자유 대화 + 행동 버튼 혼합 입력
- 감독자 모드 패널

### 처음에는 빼도 되는 것

- 실시간 다중 사용자 동기화
- 복잡한 전투
- 세밀한 이동 시뮬레이션
- 음성, 3D, 애니메이션
- 파인튜닝, RLHF

## NPC 엔진 최소 구조

각 NPC는 아래 상태를 가진다.

- persona: 성격, 말투, 가치관, 금기
- relationship: 플레이어 및 주요 NPC와의 친밀도, 신뢰, 긴장도
- memory_short: 최근 대화, 최근 사건, 직전 감정 변화
- memory_long: 반복 패턴, 중요한 사건, 플레이어에 대한 인상
- goal: 지금 원하는 것
- emotion: 현재 감정 상태
- action_policy: 어떤 상황에서 어떤 행동을 우선하는지

응답 생성 흐름은 아래처럼 단순하게 시작한다.

1. 입력 해석
2. 관련 기억 검색
3. 현재 목표와 감정 계산
4. 후보 행동 생성
5. 발화/행동 결정
6. 결과를 기억과 관계도에 반영

## 이 프로젝트의 매력 포인트

- 단순 챗봇이 아니라 세계 안에서 반응하는 NPC를 보여줄 수 있다.
- 기억과 관계 변화가 누적되어 재방문 동기가 생긴다.
- 감독자 모드를 통해 기술 시연과 사용자 경험을 동시에 만족시킬 수 있다.
- 이후 다중 사용자 확장도 자연스럽다.

## 첫 구현 우선순위

1. 허브 UI와 NPC 선택 플로우
2. NPC 상태/기억 데이터 구조
3. 대화 + 행동 버튼 인터페이스
4. 관계 변화와 퀘스트 상태 반영
5. 감독자 모드 패널

## 성공 기준

- 사용자가 같은 NPC와 두 번 대화했을 때 기억 반영 차이를 체감할 것
- 한 NPC와의 상호작용이 다른 NPC 반응에 영향을 줄 것
- 자유 대화만이 아니라 목표 지향 상호작용이 가능할 것
- 감독자 모드 없이도 재미가 있고, 감독자 모드에서는 엔진 동작이 설명 가능할 것
