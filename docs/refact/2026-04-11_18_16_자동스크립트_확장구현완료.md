## 실행한 검증

정적 검증:

```bash
node --check scripts/_episode-cli-helpers.mjs
node --check scripts/collect-episodes.mjs
node --check scripts/replay-eval-cases.mjs
npm run typecheck
npm run build
```

collector dry-run:

```bash
node scripts/collect-episodes.mjs \
  --strategies blame_supervisor,blame_director,self_defense \
  --strategy-mode weighted-random \
  --strategy-weights blame_supervisor:3,blame_director:1,self_defense:1 \
  --seed demo-seed \
  --max-episodes 4 \
  --dry-run
```

collector 최소 실행:

```bash
node scripts/collect-episodes.mjs \
  --base-url http://localhost:3000 \
  --strategy blame_supervisor \
  --max-episodes 1 \
  --output data/evals/collector-extended-summary.jsonl \
  --aggregate-output data/evals/collector-aggregate.json \
  --verbose
```

replay 최소 실행:

```bash
node scripts/replay-eval-cases.mjs \
  --base-url http://localhost:3000 \
  --cases scripts/eval-cases/sample-eval-cases.json \
  --case-id supervisor_resolution_export_case \
  --output data/evals/replay-extended-summary.jsonl \
  --verbose
```

## 아직 남은 것

- state isolation 없는 병렬 수집은 여전히 unsafe
- weighted-random 결과 품질을 위한 strategy pool 외부 설정 파일 지원은 아직 없음
- rationale / selectedAction expectation은 구현됐지만 샘플 케이스는 아직 export/resolution 쪽 위주로만 검증함
- 실제 30~100판 수집 실행은 여전히 이번 구현 범위 밖
