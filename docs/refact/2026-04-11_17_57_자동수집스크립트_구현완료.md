## 구현 내용 요약

### `scripts/collect-episodes.mjs`

- `POST /api/reset` 후 preset strategy 또는 turn file 기반으로 episode를 재생한다.
- 각 턴마다 `POST /api/interact`를 호출한다.
- resolution 도달 시 즉시 종료한다.
- episode summary를 stdout과 JSONL 파일에 남긴다.
- `free_text` 턴의 pressure movement 요약을 별도로 기록한다.
- 기본 `--max-episodes` 값은 `1`이다.
- 기본적으로 `10`판 초과 실행은 막고, 필요 시 `--allow-large-run`으로만 해제한다.
- `--dry-run`에서는 API 호출 없이 계획한 turn sequence만 출력한다.

지원 preset strategy:

- `blame_supervisor`
- `blame_director`
- `self_defense`
- `confession_recovery`
- `stall_and_shift`

### `scripts/replay-eval-cases.mjs`

- eval case JSON/JSONL 파일을 로드한다.
- case마다 `POST /api/reset` 후 turn sequence를 재생한다.
- expectation 검사 결과를 `pass / fail / warning`으로 출력한다.
- actual target, impact tags, pressure movement, retrieved knowledge/memory, resolution 상태를 요약한다.
- 결과를 JSONL로도 저장할 수 있다.

지원 expectation:

- `mustMovePressure`
- `expectedTargetNpcId`
- `expectedImpactTagsAnyOf`
- `expectedImpactTagsAllOf`
- `minKnowledgeRetrieved`
- `minMemoriesRetrieved`
- `mustResolveByRound`
- `expectedResolutionType`

## 테스트에 사용한 명령

정적 검증:

```bash
npm run typecheck
npm run build
```

collector dry-run:

```bash
node scripts/collect-episodes.mjs --strategy blame_supervisor --dry-run
```

collector 최소 실행:

```bash
node scripts/collect-episodes.mjs \
  --base-url http://localhost:3000 \
  --strategy blame_supervisor \
  --max-episodes 1 \
  --output data/evals/collector-test-summary.jsonl \
  --verbose
```

```bash
node scripts/replay-eval-cases.mjs \
  --base-url http://localhost:3000 \
  --cases scripts/eval-cases/sample-eval-cases.json \
  --output data/evals/replay-test-summary.jsonl
```

## 주의사항

- 현재 API는 단일 파일 기반 world state를 사용하므로 collector/replay를 동시에 병렬 실행하면 서로의 `reset`과 `interact`가 간섭할 수 있다.
- 실제 대량 수집은 이번 세션 범위 밖이라 수행하지 않았다.

## 아직 남은 확장 포인트

- strategy별 turn variation과 weighted sampling 추가
- collector 결과의 CSV export 또는 richer aggregate summary 추가
- replay expectation에 inspector rationale / selectedAction / export path 검사 추가
- flaky case 추적을 위한 retry / timeout / latency 로그 추가
- episode 수집을 위한 parallelism을 도입하려면 world state 분리 또는 isolated instance 전략이 먼저 필요
